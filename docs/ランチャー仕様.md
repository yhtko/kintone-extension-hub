# 依頼タイトル

サイドパネル・ランチャーに「アイコン選択＋カテゴリ分け」を追加

# 目的

ショートカットを**視覚的に識別**しやすくし、**業務カテゴリで整理**できるようにする。
既存の「ピン留め最上段」「検索→即開く」体験は維持。

# スコープ

* 対象：`popup.html / popup.css / popup.js`（サイドパネル側UI）、`options.html / options.js`（設定画面）
* 既存の件数取得（RUN_IN_KINTONE → COUNT_BULK）、ピン留め、検索、編集ダイアログは流用

---

## データモデル変更（storage）

`kintoneFavorites` の1要素に以下2つを追加：

```ts
type Favorite = {
  id: string;
  label: string;
  url: string;
  host: string;
  appId: string;
  viewIdOrName?: string;
  query?: string;
  pinned: boolean;
  order: number;
  // NEW:
  icon?: string;        // 例: "clipboard", "boxes", "calendar", "truck"
  category?: string;    // 例: "生産", "品質", "出荷", "管理"
};
```

* 既存データは `icon`/`category` が未定義でも動作するように**デフォルト空文字**扱いにする。

---

## アイコン仕様

* **Lucide**（軽量SVG）を使用。サイドパネルと設定画面で同じ名称を使う。
* 表示：ショートカット行・ボタン内に 16–18px のSVGを左に表示。テキストは省略可（2行ボタンなら下段に小さく）。
* 最低セット（候補）：`clipboard`, `file-text`, `package`, `box`, `truck`, `factory`, `wrench`, `calendar`, `list-checks`, `search`, `chart-bar`, `receipt`, `users`, `settings`, `bookmark`, `star`.

> 実装方法：`<span class="lc" data-icon="clipboard"></span>` を置き、JSで `innerHTML = lucide.icons[name].toSvg({width:16,height:16})`。

---

## カテゴリ表示仕様（ランチャー）

* ランチャー（サイドパネル）で**カテゴリごとの見出し**を表示。並び順は以下の優先順位：

  1. **ピン留めセクション**（常に最上段。従来通り）
  2. **カテゴリごとのグループ**（見出し `<h3>`：カテゴリ名）
  3. **カテゴリ未設定**は「その他」

* 検索時は**カテゴリ見出しを維持**しつつ、該当しない行は `display:none`。完全に空になったカテゴリ見出しは非表示。

## UI変更

### A. サイドパネル（popup）

1. **ショートカット行のテンプレ**

```html
<li class="kp-item" data-id="...">
  <div class="kp-left">
    <span class="kp-ico lc" data-icon="clipboard"></span>
    <div class="kp-title-txt">QB入力</div>
    <div class="kp-sub">yahat…kintone.com · view:6384656</div>
  </div>
  <div class="kp-right">
    <div class="badge">—</div>
    <button class="kp-open">開く</button>
    <button class="kp-menu">…</button>
  </div>
</li>
```

2. **カテゴリ見出し**

```html
<section class="kp-section">
  <h3>生産</h3>
  <ul class="kp-list" data-category="生産"></ul>
</section>
```

3. **描画ロジック**

* `splitPins(favs)` → `[pins, normals]`
* `groupByCategory(normals)` → `{ "生産":[…], "品質":[…], "その他":[…] }`
* 各ULに行DOMを差分描画（既存のバッジ更新ロジックはそのまま適用）

4. **CSSポイント**

* `.kp-ico` 左寄せ、16–18px。行の高さは既存維持。
* 見出し `<h3>` は余白最小（上下8px）で省スペース。

### B. 設定画面（options）

1. **登録/編集フォームに2項目追加**

* アイコン選択：

  * テキスト入力＋プレビュー、またはプルダウン（候補15〜20個）
  * 選択すると右にSVGプレビュー
* カテゴリ入力：

  * テキスト入力（サジェスト：既存カテゴリ一覧）
  * 空なら「未設定」

2. **編集ダイアログ**にも同項目を追加。保存時に `icon`/`category` を反映。

---

## 検索・キーボード操作（既存を踏襲）

* `Enter`：先頭ヒットを開く
* `↑/↓`：選択移動（カテゴリ見出しは飛ばす）
* `Esc`：検索クリア
* 検索は label/url/host/category を対象に含める

---

## バッジ（既存流用）

* ホストごとに `COUNT_BULK` を送信 → `.badge` 更新
* 未許可ホストは `—` 表示＋titleで案内
* 取得中は `.loading` で `…`

---

## 受け入れ条件（DoD）

* 既存のショートカットに対して**アイコン・カテゴリが設定可能**
* ランチャーで**カテゴリ見出しごとにグルーピング**されて表示
* 検索時、**カテゴリを保ったまま**該当行のみ表示
* ピン留めは**最上段**で従来通り動作
* アイコンが**SVGで表示**され、ライト/ダークで視認性良好
* 既存データ（icon/category未設定）でも崩れない

---

## 備考

* Lucideの読み込みは `popup.html` と `options.html` の両方に。
  例：`<script src="https://unpkg.com/lucide@latest/dist/umd/lucide.js"></script>`
* 既存CSSに `.kp-ico`（横8pxマージン）とカテゴリ見出しの余白微調整を追加。
